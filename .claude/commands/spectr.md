## Stage 1 — Frontend Analysis

Look at every image attached to this conversation. These are deduplicated frames from a screen recording of **{REFERENCE_APP}**.

For each distinct screen, produce a section in this exact format:

```
## Screen: [Name]
**Purpose:** What this screen does for the user
**Route:** Suggested route, e.g. /home, /profile/:id, /settings/notifications
**Layout:** Overall structure (tabs, lists, cards, grids, modals, etc.)
**Components:** Every visible UI element — name, type, content, and tap/click behavior
**Data Shown:** What data is displayed and its shape (list of X, single object, etc.)
**Navigation:** What screens are reachable from here and how (tab bar, back button, link, etc.)
**States:** Any loading, empty, error, or conditional states visible
**UX Notes:** Animations, gestures, swipe behaviors, notable edge cases
```

Rules:
- Merge frames that show the same screen (scroll position, loading state, etc.)
- Be exhaustive — a developer reading this should never need to guess at anything
- No preamble or closing remarks — screen specs only
